The 3n + 1 problem: a rule too simple to solve

interactive — drop in any number verified — 31 checks + 23 cross-checks sourced — proven · cited · open, flagged

Take any whole number. If it’s even, halve it; if it’s odd, triple it and add one. Repeat. Every number anyone has ever tried — and that now means all of them up to about 2.36×1021 — eventually crashes to 1. Nobody can prove it always does. This is the Collatz conjecture. Below you can watch the chaos, then meet the exact, checkable skeleton hiding underneath it.

1The rule nobody can tame

The whole problem fits on a napkin. Start with a positive integer n and apply the Collatz map over and over:

C(n) = n / 2 if n is even C(n) = 3n + 1 if n is odd

The values jump around like hailstones in a thundercloud — up on the odd steps, down on the even ones — before (so far, always) tumbling to the cycle 1 → 4 → 2 → 1. The conjecture is that every starting number reaches 1. It looks obvious from a few examples and has resisted proof for nearly a century.

What is not proven (this is an open problem)

By brute force, every starting value up to 271 = 2,361,183,241,434,822,606,848 (≈ 2.36×1021) has been checked to reach 1 (Barina 2025; the previous record was 268, Barina 2021). But no proof covers all numbers. Paul Erdős reportedly said “Mathematics is not yet ready for such problems,” and offered $500 for a solution. Everything in this page is either proven by code, cited from the literature, or flagged open. The conjecture itself stays open.

2The most stubborn numbers

Most numbers fall quickly. A few put up a fight, and mathematicians track the record-breakers: as you scan upward, which start takes the most steps so far, and which one climbs the highest?

The famous small example is 27. Its neighbours settle in a couple of dozen steps; 27 instead climbs all the way to 9,232 and needs 111 steps to reach 1. The most stubborn start below a million is 837,799: 524 steps, peaking at 2,974,984,576 — nearly three billion, about 3,550× its own size. It is the last of 44 total-stopping-time records below 106 (OEIS A006877). Drag the scan limit below and these tables fill themselves — computed live, never hard-coded.

3The hidden skeleton: Terras’s parity bijection

Here is where the chaos acquires an exact backbone. First switch to the accelerated (Syracuse) map: since 3n+1 is always even when n is odd, fold that forced halving into one step.

T(n) = n / 2 if n is even T(n) = (3n + 1) / 2 if n is odd

Now forget the values and keep only the parity at each step: write a 0 when the even rule fired and a 1 when the odd rule fired. After k steps you have a length-k string of bits — the parity vector of n.

Terras’s theorem (1976). That parity vector depends only on n mod 2k, and the map (n mod 2k) → (its k-bit parity vector) is a bijection ℤ/2k → {0,1}k: each of the 2k possible bit-patterns is produced by exactly one residue class. It is the cleanest piece of order in the whole problem — and it holds for the accelerated map T but fails for the raw map C. Flip the toggle to watch raw C collide.

The affine form that makes it exact

The bijection is really a statement about an invertible affine map. After k accelerated steps, 2k·Tk(n) = 3a·n + c, where a is the number of odd steps so far and c is an integer fixed by the parity pattern (hence by n mod 2k). The readout above computes this identity live for n = 27. Because 3a is odd and 2k is a power of two, the map is invertible mod 2k — which is exactly why every pattern has a unique residue. cited Terras (1976); Lagarias (1985).

4Why almost every number falls

The bijection does real work. Group the numbers by their k-bit parity vector, and ask: after k accelerated steps, has a number dropped below where it started? One accelerated step multiplies by 3/2 on an odd bit and by 1/2 on an even bit, so after k steps with a odd bits the value is scaled by 3a / 2k (times a bounded additive wobble). When 3a < 2k, that class has provably descended.

How many classes are safe? Terras’s exact count: among the 2k residues mod 2k, the number with exactly a odd steps is precisely C(k, a) — the binomial coefficient. So the odd-step count is distributed exactly Binomial(k, ½). The threshold 3a<2k is just a < k·log 2 / log 3 ≈ 0.6309 k, comfortably to the right of the mean k/2 — so the descending fraction climbs toward 1 (slowly) as k grows. This is the engine behind Terras’s density-1 theorem.

What “almost every” really means here

The sieve proves a density statement, not the conjecture. Terras (1976) — independently Everett (1977) — turned this counting into a theorem: the set of n whose trajectory eventually drops below n has natural density 1. Krasikov–Lagarias (2003) made it quantitative: at least x0.84 of the integers up to x reach 1. And Tao (2019/2022) proved that almost all orbits attain almost bounded values (in logarithmic density). None of these rule out a rare escaper — the conjecture stays open. cited

5Why “3” and “+1” matter

If the rule is so simple, why is it so finely balanced? Change one symbol and the behaviour flips. The same drift argument tells you why: each odd accelerated step multiplies by about q/4 for the map qn+1 (the ×q is followed, on average, by two halvings). For q = 3 that is 3/4 < 1 — expect shrinkage, so falling-to-1 is plausible. For q = 5 it is 5/4 > 1 — expect growth, so most orbits should run away.

Re-computed here (two independent ways where it matters)

  • The raw map C and accelerated map T, trajectories, total stopping time and altitude — all on exact big integers, so nothing overflows. 27 → 111 steps, peak 9,232, reproduced live.
  • Terras’s parity bijection: the residue → parity-vector map is a bijection ℤ/2k → {0,1}k for the accelerated map T (checked exhaustively k = 1…14) and fails — the same pattern collides — for the raw map C (k = 2…8). The affine identity 2kTk(n)=3an+c checks out.
  • The descent sieve: the empirical odd-step tally over residues equals the Binomial(k,½) coefficients exactly; the provably-descending fraction rises 0.8125 → 0.8281 → 0.8491 → 0.8684 → 0.8852 (k = 5,10,15,20,25); the only survivors mod 32 are {7, 15, 27, 31}.
  • Records match OEIS A006877/A006878 (steps) and A006884/A006885 (altitude): the most stubborn start below 106 is 837,799 → 524 steps, peak 2,974,984,576 — the last of 44 records.
  • Sibling maps: 3n−1 cycles {5,14,7,20,10} and the 18-long one through 17; 5n+1 has the cycle through 1 and a distinct one through 13, while 7 climbs with no cycle found; the rational cycle 151/47 is rebuilt from its parity pattern.

Cited, not re-proved

  • The conjecture itself is open. The verification that all n ≤ 271 (≈ 2.36×1021) reach 1 is Barina’s GPU computation (2025), not ours.
  • Terras (1976) / Everett (1977): the density-1 stopping-time theorem — we recompute its sieve, not the theorem.
  • Tao (2019/2022), Forum of Math, Pi: almost all orbits attain almost bounded values (logarithmic density).
  • Krasikov–Lagarias (2003): at least x0.84 of integers up to x reach 1.
  • Cycle bounds: Eliahou (1993) — any nontrivial cycle has ≥ 17,087,915 elements (his 1993 figure; the bound climbs with the verification limit, reaching ≈ 2.2×1011 by 2025); Hercher (2023) — no m-cycle for m ≤ 91.
  • The q/4 drift “why 3 not 5” argument is a heuristic (proven only for the 2-adic measure), not a proof about integers.
